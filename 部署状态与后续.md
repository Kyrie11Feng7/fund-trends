## ⚠️ 待补：每日自动更新 workflow（推送失败）
**原因**：GitHub 对 `.github/workflows/` 路径的写操作要求 token 拥有 **`workflow` scope**（普通 `repo` 权限不够，API 统一返回 404）。你提供的 PAT 只含 `repo` 权限，所以 `daily-update.yml` 未能推上去。

**二选一补救**：
- **方案 A（推荐，最省事）**：你已在 GitHub 网页打开了仓库 → 直接手动创建文件 `.github/workflows/daily-update.yml`，把工作区里这个文件的内容粘进去提交即可。内容就在 `D:\workbuddy工作空间\2026-07-18-23-52-41\.github\workflows\daily-update.yml`。
- **方案 B**：去 github.com/settings/tokens 重新生成一个**勾选了 `workflow` scope** 的 PAT，发我，我直接用 Git Data API 推上去。

## 🔧 关键前提：每日"真实刷新"还差一步
即使 workflow 就位，**当前 `pipeline.js` 不会真的改变数据**——它的行情解析是占位的（`fetchTechnical` 拉不到就回退上一版数值）。要让"每日自动更新"名副其实，需要：
1. 接入真实行情/净值数据源（如天天基金、东方财富接口，或你原有的 AI 抓取链路）；
2. 完善 `pipeline.js` 里 `fetchTechnical` 的解析，把真实技术面写进 `fund_signals.json`。

这一步做完，每日定时任务才会产生新的信号数据。在此之前，每日任务会"跑但数据不变"。
