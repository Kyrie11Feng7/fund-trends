数据口径与方法论

1. 更新时间

数据以交易日收盘后 T+1 自动同步；本页基准为 2026-07-17。每日管线由 GitHub Actions 定时触发（见仓库 .github/workflows）。

2. 分组归因规则

  • G1 纳指100/美股成长：底层 QQQ。
  • G2 信息科技/半导体/AI：底层 XLK + SOXX。
  • G3 全球新能源车：海外 NEV 指数。
  • G4 A股科创主题：科创板/创业板，与美股独立分化。
  • G5 新兴/亚洲/混合：MSCI EM / ACWI，美股科技低 beta 版。

3. 前瞻信号算法

oversold = rsi6 < 35 且 close <= bollLower
belowShort = close < ma20 且 macd < 0
strongTrend = adx > 25

短期：oversold&!strongTrend→超卖待反弹；oversold&strongTrend→超卖但趋势强；!belowShort&macd>0→偏强；其他→弱势下行
中期：close>ma120&close>ma250→长牛结构未破；close>ma250→中性；其他→趋势转弱

4. 净值曲线与回测

净值曲线读取 fund_nav.json（当前为示意序列）；回测读取 backtest.json，策略为「超卖待反弹」买入持有 5 日。两者均为示意，真实数据由 pipeline.js 生成。

5. 局限说明

实时 quote 接口在部分数据源不可用，使用 K 线收盘价替代；纯场外 QDII 使用底层分组代理信号；净值与回测当前为 DEMO，需接入真实历史数据。

本方法论为技术趋势标签，非买卖建议。
